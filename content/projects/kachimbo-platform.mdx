## Problema

Los estudiantes preuniversitarios necesitan **contenido dinámico, simulacros de examen
y seguimiento de su progreso** en un solo lugar. Muchas soluciones son fragmentadas o
dependen de infraestructura de terceros con costos altos y poco control.

Kchimbo nace como **plataforma EdTech** integrada, desplegada sobre infraestructura
propia para tener control total de rendimiento, datos y escalabilidad.

## Arquitectura

El cliente llega por Nginx (TLS), que enruta a la aplicación Next.js corriendo en
contenedor Docker sobre la VM de Oracle Cloud, con PostgreSQL como base de contenido,
progreso y usuarios.

<ArchitectureDiagram preset="kchimbo" />

- **Compute**: VMs con Linux Ubuntu para la aplicación y los servicios
- **Reverse proxy**: Nginx para TLS y entrega de assets
- **Contenedores**: Docker para consistencia entre entornos y despliegues repetibles
- **Web**: Next.js (SSR, rutas dinámicas)
- **Datos**: PostgreSQL para contenido, tracking de progreso y usuarios

## Decisiones y tradeoffs

- **Infraestructura propia en vez de PaaS** → control total de datos, rendimiento y
  costos, a cambio de operar la VM (parches, backups, monitoreo).
- **Docker** → despliegues consistentes y reproducibles; el tradeoff es la capa extra
  de build/orquestación de contenedores.
- **Next.js con SSR** → buen SEO y primer render rápido para contenido educativo, a
  costa de necesitar un runtime Node en el servidor (no solo estáticos).
