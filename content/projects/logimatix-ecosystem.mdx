## Problema

Las empresas pierden horas en tareas repetitivas: gestión de datos, respuestas a
clientes, coordinación entre sistemas que no se hablan entre sí. Cada proceso manual
es un cuello de botella y una fuente de errores.

Logimatix nace para resolverlo con **automatización inteligente operada sobre
infraestructura propia** — no un script suelto, sino una plataforma.

## Arquitectura

El sistema corre íntegramente sobre **Oracle Cloud**. Un trigger de negocio
(formulario, email o webhook) entra por Nginx, lo procesa el orquestador n8n, y desde
ahí se ramifica a la base de datos, a las APIs de IA y a las integraciones externas.

<ArchitectureDiagram preset="logimatix" />

- **Compute**: Ubuntu VMs administradas directamente (hardening, actualizaciones, monitoreo)
- **Reverse proxy**: Nginx como capa de entrada, TLS y enrutamiento
- **Orquestación**: n8n self-hosted para los flujos
- **Datos**: PostgreSQL como base transaccional y registro de ejecuciones
- **Inteligencia**: APIs ML/LLM integradas a los flujos para clasificación y generación
- **Integraciones**: CRM, pagos (Stripe/PayPal) y notificaciones (email/SMS)

## Decisiones y tradeoffs

- **Self-hosted sobre Oracle Cloud en vez de SaaS** → control de datos, costo
  predecible y libertad para integrar APIs LLM sin límites de plataforma. El tradeoff
  es asumir la operación (parches, monitoreo, backups) en lugar de delegarla.
- **n8n como orquestador** → flujos versionables y reutilizables, con lógica custom en
  Python donde el nodo estándar no alcanza. Evita reescribir integraciones desde cero.
- **PostgreSQL como única fuente de verdad** → cada ejecución queda registrada para
  trazabilidad y depuración, a cambio de mantener el esquema y las migraciones a mano.
